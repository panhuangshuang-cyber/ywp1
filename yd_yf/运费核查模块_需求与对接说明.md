## 1. 整体架构（推荐最终形态）

```
┌─────────────────────────────┐
│   主流程脚本（或 notebook）   │
│  - 加载 SKU 清单             │
│  - 调用各模块                │
│  - 聚合 + 计算差异           │
│  - 生成独立报告              │
└────────────┬────────────────┘
             │
    ┌────────┼────────┬──────────────┬──────────────┐
    ▼        ▼        ▼              ▼              ▼
yd_order_api.py   export_dingtalk_fbm_cost_v7.py   sku_alias.py   需核查运费SKU清单.xlsx
(API拉取+清洗)     (成本数据生成/提供)            (别名映射)      (SKU白名单)
```

### 各模块职责（严格分离）

| 模块 | 文件 | 职责 | 输入 | 输出 |
|------|------|------|------|------|
| **订单数据** | `yd_order_api.py` | API拉取 + 数据清洗 | 日期范围 | 干净的订单 DataFrame（已清洗） |
| **成本数据** | `export_dingtalk_fbm_cost_v7.py` | 提供 FBM 尾程运费 | - | `SKU` + `FBM_尾程运费` DataFrame |
| **别名映射** | `sku_alias.py` | SKU ↔ 别名 双向映射 | SKU列表 | 映射表 / 主SKU转换函数 |
| **SKU清单** | `需核查运费SKU清单.xlsx` | 需核查的SKU白名单 | - | SKU列表 DataFrame |
| **主流程** | `freight_check_main.py`（建议新建） |  orchestrate 各模块 + 聚合 + 报告 | 以上模块 | 独立报告 + 可返回 DataFrame |

**核心原则**：**模块间只通过 DataFrame / 函数调用传递数据，严禁使用 xlsx 作为中间媒介**（最终报告除外）。

## 2. 详细模块说明

### 2.1 `yd_order_api.py`（订单数据 + 清洗）
- 封装运德 API 调用（签名、分页、代理处理）
- **已在内部完成数据清洗**（按原操作说明：删废弃、运费≤0、数量≠1 等）
- 对外只暴露干净数据接口，例如：
  ```python
  from yd_order_api import get_clean_orders
  df_orders = get_clean_orders(days=90)
  ```

### 2.2 `export_dingtalk_fbm_cost_v7.py`（成本数据）
- 推荐暴露函数：
  ```python
  from export_dingtalk_fbm_cost_v7 import get_fbm_cost_df
  df_cost = get_fbm_cost_df()   # 返回 SKU + FBM_尾程运费
  ```

### 2.3 `sku_alias.py`（别名处理）
- 管理 SKU 与别名的对应关系
- 提供转换函数，例如：
  ```python
  from sku_alias import normalize_sku, get_alias_map
  main_sku = normalize_sku(raw_sku)   # 把别名转为主SKU
  ```

### 2.4 `需核查运费SKU清单.xlsx`
- 仅作为**白名单输入**（SKU 列）
- 建议只读 `SKU` 列，不再依赖原大 Excel 的其他计算列

### 2.5 主流程脚本（建议新建 `freight_check_main.py`）
负责：
1. 加载 SKU 白名单
2. 调用 `yd_order_api` 获取干净订单
3. 调用 `sku_alias` 做 SKU 标准化（含别名）
4. 调用 `export_dingtalk_fbm_cost_v7` 获取成本
5. 按SKU聚合（含箱子 36BOX01/56BOX02/55BOX04 多列合并）
6. 计算差异并高亮
7. 生成**独立报告**

### 方式一：完整主流程（推荐）
```python
# freight_check_main.py
from yd_order_api import get_clean_orders

from sku_alias import normalize_sku, get_alias_map
import pandas as pd

def run_freight_check():
    # 1. 获取数据
    df_orders = get_clean_orders(days=90)           # 已清洗
    df_cost = get_fbm_cost_df()
    alias_map = get_alias_map()

    # 2. SKU 标准化（把别名转为主SKU）
    df_orders['SKU'] = df_orders['产品编号'].map(alias_map).fillna(df_orders['产品编号'])

    # 3. 加载SKU白名单（可选过滤）
    df_sku_list = pd.read_excel("需核查运费SKU清单.xlsx")[['SKU']]

    # 4. 聚合 + 计算（主流程自己实现）
    df_result = aggregate_and_diff(df_orders, df_cost, df_sku_list)

    # 5. 生成报告
    generate_independent_report(df_result)
```
